# ステップアップ：Claude Code CLI（任意）

アプリ内の [Code 機能](code.md) で物足りなくなったら、ターミナルで動く本格版 **Claude Code CLI** に進めます。より大きな作業・自動化・開発に向いています。**必須ではありません**ので、必要になってからで大丈夫です。

!!! info "これは上級者向けです"
    まずは Claude Desktop の [Code 機能](code.md) で十分なことがほとんどです。本章は「もっと本格的にやりたくなったら」の案内です。

## いつ CLI に進むべきか

- 大きなプロジェクトや、たくさんのファイルをまとめて扱いたい
- 決まった作業を繰り返し自動化したい
- 開発（プログラミング）を本格的に行いたい

## Windows へのインストール

!!! warning "最新の公式手順に従ってください"
    インストール方法は更新されることがあります。必ず公式ドキュメント（[https://docs.claude.com/](https://docs.claude.com/) の Claude Code 項目）で**最新の手順**を確認してください。以下は一般的な流れです。

Windows では主に次のいずれかで導入します。

- **ネイティブ版インストーラ（おすすめ）** … PowerShell から公式のインストールコマンドを実行
- **npm 経由** … Node.js（18以上）を入れてから `npm install -g @anthropic-ai/claude-code`

導入後、ターミナル（PowerShell）で次を実行して、入っているか確認できます。

```powershell
claude --version
```

!!! note "🖼 画面イメージ（後日挿入予定）"
    ターミナルで `claude` を起動した画面。

## サブスクリプション認証でログイン

開発者向けの **API キーは使わず**、契約プラン（Pro / Max）のアカウントでログインします。

1. ターミナルで `claude` を起動
2. ログインを求められたら、案内に従って **「サブスクリプション（Claude アカウント）でログイン」** を選ぶ
3. ブラウザが開くので、Claude Desktop と同じアカウントでサインインする

!!! tip
    すでに Claude Desktop にログイン済みなら、同じアカウントを使います。API キーの作成は不要です。

## 基本操作

- ターミナルで作業したいフォルダに移動し、`claude` と入力して起動
- あとは**日本語で依頼するだけ**（例：「このフォルダの README を更新して」）
- 終了は `exit` または ++ctrl+c++ を2回

### モード切替（自動モード）

CLI でも、[Code 機能と同じ複数のモード](code.md#auto-mode)（許可を確認／編集を承認／プランモード／自動モード／許可をバイパス）があります。CLI では入力中に ++shift+tab++ を押すと、モードが順番に切り替わります。

!!! info
    **自動モード**でも Claude 自身の安全確認は働きます（省かれるのは逐一の承認）。一方、**許可をバイパス**はすべての確認を飛ばすので要注意です。内容を把握できる作業・信頼できる範囲で使ってください。
