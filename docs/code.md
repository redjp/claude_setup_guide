# Code 機能を使う（Desktop 内）

上部タブの **「Code」** では、Claude Desktop の中で **コードの作成・修正やファイル操作、定型作業の自動化** などの実作業を任せられます。プログラミングだけでなく、「フォルダ内のファイルを整理する」「データを変換する」といった作業にも使えます。

!!! info "まずは Chat / Cowork で十分なことも多い"
    文章作成やデータ整理だけなら [Chat](quickstart.md) や [Cowork](cowork.md) で足ります。Code 機能は、**ファイルやコードを直接さわる作業**が必要なときの選択肢です。

## Code 機能とは / 始め方

1. 上部タブの **「Code」** を開く
2. 作業対象の **フォルダ（プロジェクト）** を指定する
3. やってほしいことを言葉で依頼する（例：「このフォルダの請求書PDFから金額を抜き出して一覧にして」）

![Code タブ下部のバー：実行場所・対象フォルダ・ブランチの指定](img/code-interface.png)

下部のバーで、**実行場所（ローカル）・対象フォルダ（プロジェクト）・Git のブランチ**などを指定し、その下の入力欄に依頼を書きます。

## モード切替（自動モード） { #auto-mode }

Code 機能では、Claude に **「どこまで自分で進めてよいか」** を切り替えられます。入力欄の左下のモード表示から、次の5つを選べます。

| モード | 動き | 向いている場面 |
| --- | --- | --- |
| **① 許可を確認**（既定） | 操作のたびに、あなたに「実行してよいか」を確認 | 初めて・大事な環境。**まずはこれ** |
| **② 編集を承認** | ファイルの編集は自動で進め、それ以外は確認 | 編集作業をテンポよく |
| **③ プランモード** | 実行はせず、**計画だけ**を立てる（読み取りのみ） | まず段取りを確認したい |
| **④ 自動モード** | 逐一の確認は省き、**Claude 自身が安全性を判断しながら**自動で進める | 内容を把握できる・信頼できる作業 |
| **⑤ 許可をバイパス** | **すべての確認を省略**して実行 | 上級者向け・**要注意** |

!!! info "「自動モード（④）」と「許可をバイパス（⑤）」は別物です"
    - **自動モード（④）** … 逐一の承認は省きますが、**Claude（エージェント）自身が安全性を確認しながら**進めます。「無条件に何でも実行する」モードではありません。
    - **許可をバイパス（⑤）** … こちらが**すべての確認を飛ばす**モードです。取り返しのつかない操作も起こり得るため、**内容を完全に把握できる場合のみ**にしてください。

!!! tip "経営者へのおすすめ"
    まずは **① 許可を確認** で様子を見て、慣れたら信頼できる作業で **④ 自動モード**。**⑤ 許可をバイパス は基本的に使わない**のが安全です。

![入力欄の左下から開く、5つのモード切り替えメニュー](img/code-modes.png)

## 基本の使い方と活用例

- 「この CSV を、月別の集計表に変換して」
- 「フォルダ内のファイル名を、`日付_案件名` の形式にそろえて」
- 「議事録のテキストから、ToDo を抜き出して一覧ファイルにして」

!!! tip "もっと本格的に使うなら"
    Code 機能で物足りなくなったら、ターミナルで動く本格版 **[Claude Code CLI](cli.md)** に進めます。
